## UI polish workflow

- For final frontend polish, prioritize these skills when relevant:
  - `ui-ux-pro-max`
  - `web-design-guidelines`
  - `frontend-design`
  - `taste-skill`
- If the codebase uses `shadcn/ui`, prefer existing `shadcn/ui` patterns and composition.
- If motion exists, keep it subtle, fast, and purposeful.
- Preserve functionality and data flow.
- Prefer black, neutral, professional styling unless the repository already defines a different design system.
- Always audit first, then plan, then implement, then review accessibility.
